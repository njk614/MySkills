# 集成说明

## 目标

保持两套输出同时成立：

1. 面向用户的可读结果（中文或英文）
2. 面向执行接口的原始编码指令

## 原始响应分段格式

兼容格式如下：

```json
{
  "message": "根据最优策略，已经为您规划如下执行计划：\n1、场景复位"
}
```

返回格式：

- 执行层将直接输出一个固定的 JSON 对象给调用方（前端/宿主），不再输出分隔标记。示例见上文。

## HTTP 执行请求

### 请求方法

- `POST`

### 请求路径

- `/v1/location/SendInstruction`

### 请求头

- `Content-Type: application/json`
- `Accept: text/plain`

### 请求体

```json
{
  "token": "场景 token",
  "jsonData": "instruction_order$&query$&plan_text"
}
```

### 字段解释

- `instruction_order`：带编码的执行指令串，如 `A09` 或 `A36：告警信息：当前$A38：告警信息选中`
- `query`：原始用户问题
- `plan_text`：面向用户的展示文本，不带编码，可为中文或英文

## 关键原则

### 展示文本

必须：

- 不带 `A/B/C/D` 编码
- 根据用户语言返回中文或英文可读文本
- 与原工作流展示风格一致

### 执行文本

必须：

- 保留原始指令编码
- 可直接拼入 `jsonData`

归一化规则：

- 无参数固定指令与固定枚举值指令发送给孪易时，使用中文标准语义。
- 名称类参数指令保留查询层匹配返回的原始名称，匹配为中文就发中文，匹配为英文就发英文。
- `C02` 与 `D01` 的内容部分允许保留当前用户语言。

## 示例

### 示例 1：场景复位

- `instruction_order`: `A09`
- `query`: `场景复位`
- `plan_text`: `根据最优策略，已经为您规划如下执行计划：\n1、场景复位`
- `jsonData`: `A09$&场景复位$&根据最优策略，已经为您规划如下执行计划：\n1、场景复位`

### 示例 2：告警截图

- `instruction_order`: `A36：告警信息：当前$A38：告警信息选中$A39：告警截图：打开`
- `query`: `查看告警触发截图`
- `plan_text`: `根据最优策略，已经为您规划如下执行计划：\n1、告警信息：当前\n2、告警信息选中\n3、告警截图：打开`
- `jsonData`: `A36：告警信息：当前$A38：告警信息选中$A39：告警截图：打开$&查看告警触发截图$&根据最优策略，已经为您规划如下执行计划：\n1、告警信息：当前\n2、告警信息选中\n3、告警截图：打开`

### 示例 3：设置视频显示模式

- `instruction_order`: `E02：筛选：设置显示模式，3×3`
- `query`: `设置显示模式为3×3`
- `plan_text`: `根据最优策略，已经为您规划如下执行计划：\n1、筛选：设置显示模式，3×3`
- `jsonData`: `E02：筛选：设置显示模式，3×3$&设置显示模式为3×3$&根据最优策略，已经为您规划如下执行计划：\n1、筛选：设置显示模式，3×3`

## 历史上下文

如宿主系统支持会话状态，保存：

- `historyUser`: 最近问题与原始方括号指令内容
- `historyInter`: 工具调用名与工具返回结果
- `sceneInfo`: 最近场景配置
- `tokenJudge`: 最近一次 token
