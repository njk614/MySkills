# 指令规则

## 目标

将用户自然语言转换为两份结果：

1. 执行用指令串：保留 `A/B/C/D` 编码，用于发送执行 HTTP 请求。
2. 展示用中文文本：不保留编码，只保留用户可理解的动作说明。

- `A14` → `时间轴：暂停`
- `A20` → `图层全部显示`
- `A21` → `图层全部隐藏`
- `A31` → `停止演示`
- `A32` → `暂停演示`
- `A33` → `上一步演示`
- `A34` → `下一步演示`
- `A35` → `重新演示`
- `A36` → `告警信息：当前`

## 查询类指令

- `D01` 用于查询类结果。
- 展示时不再保留 `D01` 编码。
- 直接输出"为您查找到相关内容如下：..."的自然语言文本。

## 展示文本规则

### 原则

- 用户看到的结果不带编码。
- HTTP 执行的 `instruction_order` 保留编码。
- 不要把执行编码直接展示给用户。

### 示例

- `A09` → 展示 `场景复位`
- `A08：场景旋转：开始` → 展示 `场景旋转：开始`
- `A01：功能切换：告警` → 展示 `功能切换：告警`
- `B07：打开灯：1F走廊灯` → 展示 `打开灯：1F走廊灯`
- `C02：主题生成：园区能耗分析` → 展示 `主题生成：园区能耗分析`

## 多指令规则

- 多条执行指令使用 `&` 连接。
- 展示时按序号拼接：
  - `1、...`
  - `2、...`
- 非查询类统一前缀：
  - `根据最优策略，已经为您规划如下执行计划：`

## 未匹配规则

遇到以下情况时，不要伪造结果：

- 名称类参数未匹配到。
- 状态类参数未判断出。
- 只返回裸编码但该指令必须带参数。

此时应明确返回失败原因或候选信息。

# 视频监控指令规则（E 系列）

## 目标

将用户自然语言转换为两份结果：

1. 执行用指令串：保留 `E` 编码，用于发送执行 HTTP 请求。
2. 展示用中文文本：不保留编码，只保留用户可理解的动作说明。

## 无参数固定指令（E 系列）

以下指令本身不需要额外参数，可直接转换为固定中文展示文本：

- `E05` → `视频：视频上一页`
- `E06` → `视频：视频下一页`
- `E08` → `视频：下一个视频`
- `E09` → `视频：上一个视频`
- `E10` → `视频：第一个视频`
- `E11` → `视频：末一个视频`
- `E17` → `事件：下一个事件`
- `E18` → `事件：上一个事件`
- `E19` → `事件：第一个事件`
- `E20` → `事件：末一个事件`
- `E22` → `回放：暂停`
- `E23` → `回放：播放`
- `E28` → `单路云台：左转`
- `E29` → `单路云台：右转`
- `E30` → `单路云台：抬头`
- `E31` → `单路云台：低头`
- `E32` → `单路云台：拉近`
- `E33` → `单路云台：拉远`

## 必须带参数的指令（E 系列）

以下指令不能只返回裸编码；必须返回完整内容再生成展示文本：

- `E01` 范围筛选（中心点坐标 + 范围半径）
- `E02` 显示模式设置（单路、2×2、3×3 其一）
- `E03` 视频轮播（开始、停止 其一）
- `E04` 视频排序（按对象名称正序/倒序、按创建时间正序/倒序 其一）
- `E07` 视频指定页（具体页码）
- `E12` 事件列表（选中、取消 其一）
- `E13` 事件轮播（开始、停止 其一）
- `E14` 事件筛选（筛选条件）
- `E15` 事件排序（按时间正序、按时间倒序 其一）
- `E16` 选中事件（具体事件标识）
- `E21` 时间模式切换（实时、回放 其一）
- `E24` 回放跳转（目标时间点）
- `E25` 回放前进（时长）
- `E26` 回放回退（时长）
- `E27` 回放倍速（倍速值）
- `E34` 按摄像头名称筛选（摄像头名称，必须从 `get_bind_video_instance_names` 获取）
- `E35` 输出摄像头名称列表（多个名称，从 `get_bind_video_instance_names` 获取）

## 查询类指令（E35）

- `E35` 用于询问类查询（如"可控摄像头有哪些"），输出名称列表。
- 展示时不保留 `E35` 编码。
- 直接输出"为您查找到相关内容如下：…"的自然语言文本，列出所有匹配名称。

## MCP 工具调用规则（E 系列）

| 指令    | 是否需要摄像头名称 | 说明                       |
| ------- | ------------------ | -------------------------- |
| E01–E33 | 否                 | 纯逻辑指令，无需摄像头名称 |
| E34     | 是                 | 需查询摄像头实例名称       |
| E35     | 是                 | 需查询摄像头实例名称       |

调用前先检查 `history_inter` 中是否已有 `get_bind_video_instance_names` 的返回数据，有则直接使用，不重复调用。

未匹配到摄像头名称时，直接输出 `[视频中没有找到匹配的信息]`，不拼接任何指令。

## 展示文本规则（E 系列）

- `E08：视频：下一个视频` → 展示 `视频：下一个视频`
- `E34：筛选：1号摄像头` → 展示 `筛选：1号摄像头`
- `E02：筛选：设置显示模式，3×3` → 展示 `筛选：设置显示模式，3×3`
- `E12：事件：事件列表，选中` → 展示 `事件：事件列表，选中`
- `E21：时间：模式切换，回放` → 展示 `时间：模式切换，回放`
- `E32：单路云台：拉近` → 展示 `单路云台：拉近`

## 组合指令规则（E 系列）

- 查看摄像头详情：`E34：筛选：{摄像头名称}`
- 摄像头拉近：`E34：筛选：{摄像头名称}&E32：单路云台：拉近`
- 摄像头拉远：`E34：筛选：{摄像头名称}&E33：单路云台：拉远`
- 查看摄像头事件/告警：`E34：筛选：{摄像头名称}&E12：事件：事件列表，选中`

当用户只提"放大、缩小、拉近、拉远、左转、右转、抬头、低头"而不带摄像头名称时，直接输出单路云台相关指令。
