# 空间指标映射

本文档把常见用户请求映射到后端应展示的空间利用指数字段名称和范围语义。

## 当前实际返回的指标字段

- `SpaceOccupancyRate`
- `SpaceUsageFrequency`
- `PerCapitaAreaComplianceRate`
- `SpaceReuseIndex`
- `FlexibleDeskReservationFulfillmentRate`
- `MeetingRoomVacancyScore`
- `SpaceConflictResolutionRate`
- `SpaceVitality`
- `SpaceIntensity`
- `SpaceQuality`
- `SpaceUtilizationIndex`

## 用户请求与展示语义

- “空间利用情况怎么样”
  - 优先展示 `Message`、`RegionData`、`FloorData`
  - 如有需要，再展开 `Indicators`

- “固定工位区的占用率”
  - 只展示固定工位区对应结果里的 `SpaceOccupancyRate`
  - 不把同一条结果里的其他指标一并输出，除非用户继续追问

- “空间使用率 / 占用率”
  - 优先对应 `SpaceOccupancyRate`
  - 这是“被占用的平均时间比例”，不是空置率

- “会议室空置率 / 空置率评分”
  - 优先对应 `MeetingRoomVacancyScore`
  - 这是“被预定但未被实际使用的比例”换算成 0-100 分后的结果
  - 如果用户追问“怎么评出来的”，要直接解释得分公式和默认口径，不要只回指标名

- “这个楼层怎么样”
  - 对应某个 `levelName` 下的整层结果与区域结果

- “某个区域怎么样”
  - 传入 `regionName`，只展示该区域的 `RegionData`

## 范围提示

- `levelName` 是层级名称，例如 `1F`、`2F`、`B1`
- 如果用户问题没有明确层级，默认使用 `楼层20`
- `regionName` 是区域名称，例如某个固定工位区或会议室区；如果未识别到，默认传 `""`（空字符串）
- `FloorData` 为整层聚合结果，仅在 `regionName` 为空字符串时返回
- `RegionData` 为区域级结果，传入 `regionName` 后只展示这里的结果

## 时间粒度提示

- `Hour`：小时桶起始时间
- `Day`：日桶时间，固定为 `00:00:00`
- `Week`：周一 `00:00:00`
- `Month`：每月 1 号 `00:00:00`
- 如果用户问题没有明确时间范围，默认按当前小时桶查询，`startTime` 为当前小时桶起始时间，`endTime` 为当前小时桶结束时间或当前时刻

## 返回提示

- 始终优先使用后端返回的数值。
- 当用户询问原因、时间边界或粒度语义时，附上后端给出的 `Message` 和本文件中的解释。
- 如果后端返回多个区域结果，先按后端结果展示，不要自行合并或筛选。

## 指标解释知识点

### 通用计算口径

- 所有空间利用指数相关公式都来自 `docs/Spacecount/园区行业模型-空间利用指数指标模型.md`。
- 只要用户问的是“怎么算”，就按本文件直接解释，不要笼统回答“后端内部计算”。
- 可以解释指标定义、公式、权重、时间粒度、空间粒度、有效性规则，但不要在没有后端返回值时本地编造当前分数。
- 默认工作时段为 09:30:00 ≤ 时间 < 19:30:00，非工作时段不参与空间利用相关指标计算。
- 单个空间在线率低于 80% 时，该空间不计入区域聚合分母；若区域内某类有效空间数为 0，则该区域该类相关指标计 0；整层聚合时只计入有效区域，按面积加权平均。
- 时间粒度规则：
  - `SpaceOccupancyRate`、`SpaceUsageFrequency` 支持小时、日、周。
  - `PerCapitaAreaComplianceRate`、`SpaceReuseIndex`、`FlexibleDeskReservationFulfillmentRate`、`MeetingRoomVacancyScore`、`SpaceConflictResolutionRate` 只支持日、周；若界面展示小时值，按日值回填。
  - `SpaceUtilizationIndex` 支持小时、日、周、月。

### L3 指标公式

#### 空间占用率

- 对应字段：`SpaceOccupancyRate`
- 含义：统计周期内，区域内所有有效工位和会议室被占用的平均时间比例。
- 区域公式：区域占用率 = (所有有效工位的占用率\_1小时之和 + 所有有效会议室的占用率\_1小时之和) / (有效工位数 + 有效会议室数)
- 整层公式：整层占用率 = Σ(区域占用率 × 区域面积) / Σ(有效区域面积)
- 回答重点：这是“被占用的平均时间比例”，更接近使用率，不是空置率评分。

#### 空间使用频次

- 对应字段：`SpaceUsageFrequency`
- 含义：统计周期内，每个有效空间平均发生多少次“从空闲变占用”的使用事件。
- 区域公式：区域使用频次 = (所有有效工位的使用次数\_1小时之和 + 所有有效会议室的使用次数\_1小时之和) / (有效工位数 + 有效会议室数)
- 整层公式：整层使用频次 = Σ(区域使用频次 × 区域面积) / Σ(有效区域面积)
- 归一化用途：在 `SpaceVitality` 中按 2 次/空间 = 100 分做归一化。

#### 人均面积达标率

- 对应字段：`PerCapitaAreaComplianceRate`
- 含义：区域实际人均面积与标准人均面积的符合程度。
- 默认标准：标准人均面积默认 5㎡/人，可配置。
- 实际人均面积公式：实际人均面积 = 区域总面积 / 区域内工作时段最大并发人数。
- 指标公式：区域人均面积达标率 = min(100, 标准人均面积 / 实际人均面积 × 100)
- 整层公式：整层人均面积达标率 = Σ(区域人均面积达标率 × 区域面积) / Σ(有效区域面积)
- 回答重点：若实际人均面积小于等于标准，则记满分 100；不是面积越大越好。

#### 空间复用指数

- 对应字段：`SpaceReuseIndex`
- 含义：所有有工位区域内，有效工位被使用的平均频次，反映工位复用效率。
- 区域公式：区域空间复用指数 = 区域内所有有效工位的使用次数\_1小时之和 / 区域内有效工位数
- 整层公式：整层空间复用指数 = Σ(区域空间复用指数 × 区域面积) / Σ(有效区域面积)
- 归一化用途：在 `SpaceIntensity` 中按 3 次/工位 = 100 分做归一化。

#### 灵活工位预定履约率

- 对应字段：`FlexibleDeskReservationFulfillmentRate`
- 含义：灵活工位预定后被实际使用的比例。
- 区域公式：区域履约率 = 已使用的预定次数 / 总预定次数 × 100%
- 判断口径：已使用的预定次数对应空间状态为“预定使用”。
- 特殊值：如果总预定次数为 0，则该区域指标记 0 分。

### 会议室空置率评分

`MeetingRoomVacancyScore` 对应“会议室空置率得分”，不是百分比意义上的空置时间占比，而是一个 0-100 分的评分。

文档口径如下：

- 区域空置率得分 = 100 × (1 - 预定未使用次数 / 总预定次数)
- 如果总预定次数 = 0，则该区域得分 = 100 分，表示“没有预定，也就没有浪费”
- 计算基于会议室“空间状态”为“预定空置”的事件次数，以及空间预定记录中的预定总数
- 该指标只支持日/周粒度，小时值按文档回填规则处理

回答建议：

- 用户问“空置率评分怎么来的”时，优先解释“预定了但没用”的比例如何转成分数
- 用户问“使用率”时，不要误答为空置率，应该回到 `SpaceOccupancyRate`
- 如果用户明确问“会议室空置率”，可以直接给出该指标名、得分和公式说明

### 空间冲突解决率

- 对应字段：`SpaceConflictResolutionRate`
- 含义：空间预定冲突被自动或人工解决的比例。
- 区域公式：解决率 = 已解决的冲突数 / 总冲突数 × 100%
- 特殊值：如果总冲突数为 0，则指标记 100 分，表示没有冲突。
- 整层公式：整层冲突解决率 = Σ(区域冲突解决率 × 区域面积) / Σ(计入区域面积)

### L2 聚合公式

#### 空间活力

- 对应字段：`SpaceVitality`
- 公式：空间活力 = 空间占用率 × 0.5 + 空间使用频次归一化得分 × 0.5
- 归一化口径：空间使用频次归一化得分 = min(100, 实际使用频次 / 2 × 100)
- 回答重点：占用广度和流动密度各占 50%。

#### 空间集约度

- 对应字段：`SpaceIntensity`
- 语义说明：字段名是 `SpaceIntensity`，中文解释按“空间集约度”回答。
- 公式：空间集约度 = 人均面积达标率 × 0.6 + 空间复用指数归一化得分 × 0.4
- 归一化口径：空间复用指数归一化得分 = min(100, 实际复用指数 / 3 × 100)
- 回答重点：人均面积达标率权重更高，体现“节地优先，增效为辅”。

#### 空间品质

- 对应字段：`SpaceQuality`
- 公式：空间品质 = 灵活工位预定履约率 × 0.4 + 会议室空置率得分 × 0.3 + 空间冲突解决率 × 0.3
- 回答重点：分别对应用户行为、资源浪费和管理效率三部分。

### L1 总指标公式

#### 空间利用指数

- 对应字段：`SpaceUtilizationIndex`
- 公式：空间利用指数 = 空间活力 × 0.3 + 空间集约度 × 0.5 + 空间品质 × 0.2
- 默认权重含义：集约度最重要，其次是活力，最后是品质。
- 区域级：直接基于同一区域的 L2 指标计算。
- 整层级：整层空间利用指数 = Σ(区域空间利用指数 × 区域面积) / Σ(有效区域面积)
- 支持动态配置，但在未拿到新配置前，解释时默认使用 0.3 / 0.5 / 0.2。

### 典型追问回答规则

- 用户问“空间利用指数怎么计算出来的”时，必须明确回答 `SpaceUtilizationIndex = SpaceVitality × 0.3 + SpaceIntensity × 0.5 + SpaceQuality × 0.2`，再补充三项 L2 的来源。
- 用户问“空间活力怎么来的”时，说明它由 `SpaceOccupancyRate` 和 `SpaceUsageFrequency` 归一化后等权组成。
- 用户问“空间集约度怎么来的”时，说明它由 `PerCapitaAreaComplianceRate` 和 `SpaceReuseIndex` 归一化后按 0.6/0.4 组成。
- 用户问“空间品质怎么来的”时，说明它由 `FlexibleDeskReservationFulfillmentRate`、`MeetingRoomVacancyScore`、`SpaceConflictResolutionRate` 按 0.4/0.3/0.3 组成。
- 用户问“为什么空置率评分高反而更好”时，说明这是一个得分，不是空置时间百分比；分数越高表示“预定未使用”越少。
- 用户问“为什么和占用率不是一回事”时，说明占用率看的是被实际占用的时间比例，空置率评分看的是被预定但没用的浪费情况。

### 使用率与空置率的区别

- `SpaceOccupancyRate`：看的是“空间被占用的平均时间比例”，更接近使用率
- `MeetingRoomVacancyScore`：看的是“会议室预定后未实际使用”的浪费程度，分数越高表示空置越少
- 两者不是同一个指标，不能互相替代
