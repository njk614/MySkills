## 1. 文档目的

本文档用于将阶段一已确认需求转化为可开发、可排期、可验收的功能点清单和工作项明细，作为阶段二开发的直接输入。

## 2. 功能范围

### 2.1 涉及模块

- ruisi-twinioc-command-skill
- ruisi-twinioc-dataquery-skill
- ruisi-twinioc-opeationrule-skill
- ruisi-twinioc-spacecount-skill
- 孪易 Demo 前端配置
- 孪易 Demo 后端指令接收与执行链路
- Demo 场景中的 IoT 接入与 mock 适配层
- 模拟数据生成器
- MCP 接口和相关说明文档
- docs/API-DESIGN.md
- docs/openapi.yaml

### 2.2 总体目标

- 以华为园区军团会议纪要和参考图为对标输入，在现有 Demo 基础上完成轻量化的中英文改造。
- 构建一套中英文共存的 Twinioc Demo 方案，使英文用户可完成核心查询、执行、确认、规则联动和空间查询场景。
- 让中文 Demo 摆脱真实 IoT 依赖，改为以模拟数据和可控接口驱动。
- 基于整理后的中文 Demo 建立对应英文 Demo，并保证英文指令在孪易侧具备与中文一致的效果。
- 让 MCP 接口具备中英文调用和返回兼容能力，并同步提供可交付文档。

### 2.3 非目标

- 不重构孪易底层平台架构。
- 不按会议纪要参考图重做一套全新的 Demo 视觉、三维资产或交互体系。
- 不在本阶段扩展除英文外的新语种。
- 不对产品前端做全站视觉国际化改版。
- 不改变现有 A/B/C/D/E 指令编码体系。

## 4. 功能依赖关系

### 4.1 依赖原则

- 先清理中文 Demo 中真实 IoT 依赖，再建设英文 Demo。
- 先打通 dataquery 和 command 的英文主链路，再补规则、空间与 MCP 等增强能力。
- 先建立别名映射和英文指令归一化，再提高英文命中率与执行一致性。
- 先完成模拟数据适配，再做整体验收。

### 4.2 依赖关系表

| 前置功能点                   | 后置功能点 | 依赖说明                                           |
| ---------------------------- | ---------- | -------------------------------------------------- |
| F-11                         | F-12       | 英文 Demo 必须基于整理后的中文 Demo 配置           |
| F-11                         | F-13       | mock 适配需要先识别中文 Demo 中的真实 IoT 调用     |
| F-13                         | F-14       | 英文指令效果验证需要 mock 数据和 mock 操作接口支撑 |
| F-13                         | F-15       | MCP 双语接口验证需要 mock 数据返回稳定             |
| F-01                         | F-02       | 英文查询触发需要英文 skill 说明层                  |
| F-01                         | F-03       | 英文执行触发需要英文 skill 说明层                  |

## 9. 功能设计结论

- 本项目的功能设计对象应是完整 Demo，而不是只改 Skill。
- 会议纪要和参考图在功能层面主要提供展示方向与边界约束，不转换为大改版功能范围。
- P0 工作需要同时覆盖 Skill 主链路、中文 Demo 去 IoT 化、英文 Demo 配置、mock 数据适配、英文指令兼容和 MCP 双语接口。
- 阶段二应优先完成 F-11 至 F-15 与 F-01 至 F-06 的主链路能力，再补规则和空间增强能力。
