## 1. 文档目的

本文档用于明确 Twinioc Demo 中英文版改造项目要解决的核心问题、需求来源、优先级分级和对标结论，为后续功能设计、技术设计、数据设计、验收设计和阶段二开发排期提供统一输入。

## 2. 项目背景

### 2.1 业务背景

- 当前已存在一套可运行的中文 Demo，覆盖孪易产品展示、Skill 调用、场景控制、只读查询、规则匹配和空间查询等能力。
- 本次需求来源于华为园区军团 2026-03-23 会议纪要，以及对方提供的 3 张参考图，核心诉求是支撑海外推广场景下的轻量化、双语化演示。
- 会议纪要和参考图主要用于对标参考，重点是“简约、非写实、大字、信息清晰”的展示方向，而不是要求按参考图重做整套新 Demo。
- 本次改造目标不是只做 Skill 文案翻译，而是交付一个中英文都可完整演示的 Demo。
- 本次实施方式是在现有 Demo 基础上做增量修改，优先复用现有场景、调用链路和配置，不进行大规模视觉重构、三维资产重建或交互体系重写。
- Demo 级交付要求前端、后端、接口、模拟数据和 Skill 行为统一协同，否则英文 Demo 无法稳定演示。

### 2.2 当前版本现状

- 四类 skill 已具备基础中文能力，但英文触发、英文执行、英文返回仍不完整。
- 当前中文 Demo 中存在部分真实 IoT 调用依赖，不适合在标准演示环境中直接复用。
- 当前缺少独立可运行的英文 Demo 配置，英文场景无法做到低成本切换。
- 当前模拟数据生成器与传感器、操作接口绑定不完整，仍存在真实接口耦合。
- 当前孪易产品主要按中文指令链路工作，英文指令接入与中文效果一致性尚未建立。
- 当前 MCP 接口和相关文档主要按中文调用场景设计，双语兼容性不足。

## 4. 需求来源清单

- 会议纪要与参考图：来自华为园区军团会议纪要和配套参考图，提供对外演示目标和风格参考。

- 现网问题归纳：来自当前 Demo、Skill、接口、数据链路中的确定性缺口。
- Demo 交付要求：来自“必须形成可运行 Demo”这一交付边界。

### 4.3 优先级分级说明

#### P0

P0 代表本期必须完成，否则无法形成可交付的中英文 Demo：

- 面向海外客户接待、销售售前讲解、华为园区军团评审等场景，必须保证同一套 Demo 可直接完成英文查询、英文控制和英文反馈展示。
- 面向现场演示人员，必须保证英文确认、取消和执行链路完整，避免在会议室或展厅演示中因流程中断而人工接管。
- 面向国内客户汇报和现有交付场景，必须保证中文版主链路无回归，中文查询、中文控制和中文展示继续可用。
- 面向演示、非生产网络和无真实设备环境，必须完成真实 IoT 依赖隔离和模拟数据接管，保证 Demo 可复制部署。
- 面向项目交付和后续联调，必须同时提供英文 Demo 配置、双语 MCP 接口能力和统一验收口径，确保可交付、可验收、可复用。

#### P1

P1 代表建议同步纳入，以提升双语能力完整度：

- 英文规则写入、解析、匹配。
- 英文实体别名映射。
- spacecount 英文结果与建议输出。

#### P2

P2 代表可延期，不影响本期核心交付：

- 除英文外的更多语种支持。
- 更复杂的个性化语言策略。

## 8. 结论

本次中英文改造并不是简单翻译 Skill，而是一次围绕“Skill 说明层 + 语言输出层 + Demo 配置层 + 指令接收层 + 模拟数据层 + MCP 接口层”的完整 Demo 升级。

结论如下：

- 本次输入来源虽然包含华为园区军团会议纪要和两张参考图，但它们在本项目中定位为“需求来源和对标参考”，不是“重做蓝本”。
- 本项目的交付目标应定义为“中英文可运行 Demo”，而不是“中英文 Skill”。
- 当前中文资产和 Skill 资产复用比例较高，但真实 IoT 依赖、英文 Demo 配置、英文指令兼容和 MCP 双语接口仍需补齐。
- 本次改造应坚持基于现有 Demo 做轻量增量修改，优先完成双语能力、mock 化和演示闭环，不扩大到大规模界面或场景重建。
- 阶段二开发应优先完成中文 Demo 去 IoT 依赖、模拟数据适配、英文执行主链路和英文 Demo 配置，再补规则与空间等增强能力。
- 后续功能设计和开发工作项必须覆盖 Skill、孪易前后端、接口、文档和模拟数据，不然无法形成完整 Demo。
