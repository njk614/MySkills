## 1. 文档目的

本文档用于明确 Twinioc Demo 中英文版改造所需的数据范围、数据来源、字段映射、模拟数据方案、POC 验收数据以及数据工作项拆解，作为阶段二数据对接与联调的直接依据。

## 2. 数据范围

### 2.1 数据设计目标

- 为英文输入提供可映射的实体别名和命令映射数据。
- 为中文 Demo 去 IoT 化提供孪生体和接口依赖清单。
- 为英文 Demo 和模拟数据生成器提供可直接落地的样例数据集。
- 为 MCP 双语接口和阶段验收提供稳定、可复现的数据基线。
- 将会议纪要和两张参考图作为非结构化参考输入，用于约束演示风格方向和场景取舍，但不派生出整套重做数据资产。

## 5. 真实数据接入与模拟数据方案

### 5.1 接入原则

- 用于识别当前状态和梳理依赖的数据必须基于真实中文 Demo 和当前实际接口关系获取。
- 用于最终演示运行的数据应优先切换到 mock 数据源，不依赖真实 IoT。
- 英文别名映射、文案资源、规则语料、英文 Demo 配置和 MCP 双语样例可采用静态配置维护。
- 会议纪要与参考图只作为需求参考输入保留摘要，不要求沉淀为新的运行时数据源或重建整套视觉资产数据。

### 5.2 必须真实接入的数据

| 数据类型              | 原因                           | 说明                                     |
| --------------------- | ------------------------------ | ---------------------------------------- |
| 中文 Demo 孪生体清单  | 直接决定哪些对象需要保留或移除 | 必须基于当前中文 Demo 实际配置           |
| 当前 IoT 接口依赖关系 | 直接决定哪些调用需要 mock 化   | 必须从当前调用链真实梳理                 |
| 场景结构信息          | 直接影响实体存在性与命名       | 必须来自真实 scene_info 或 scene_context |
| 摄像头和孪生体实例    | 影响英文映射和英文演示命中     | 必须来自真实场景对象                     |

### 5.3 可使用模拟数据的数据

| 数据类型           | 用途               | 说明                          |
| ------------------ | ------------------ | ----------------------------- |
| 模拟传感器数据     | 支撑演示态查询     | 温度、状态等都可由 mock 生成  |
| 模拟操作回执       | 支撑演示态执行     | 操作结果、状态变化都可走 mock |
| 双语文案资源       | 支撑 locale 输出   | 可完全本地维护                |
| 英文 Demo 配置样例 | 支撑英文演示搭建   | 可本地维护固定配置            |
| MCP 双语样例       | 支撑接口文档与测试 | 可固定版本维护                |

### 5.4 推荐数据组织方式

- 中文 Demo 孪生体与 IoT 依赖清单。
- 英文实体别名映射样例表。
- 双语规则样例表。
- 指令双语映射表。
- 模拟传感器与操作数据集。
- MCP 双语请求与返回样例集。

### 5.5 数据联调与默认行为约束

- 演示环境联调默认使用 mock 数据集，除专项验证外不以 live 数据作为通过前提。
- 双语请求样例需同时提供“显式携带 locale”和“未携带 locale”的两类样例，用于验证缺省行为。
- mock 传感器数据和 mock 操作回执必须使用同一组对象主键或绑定键，避免读写状态脱节。
- 英文 Demo 配置样例必须引用整理后的中文 Demo 主体对象，不单独复制一套新的底层对象数据。

## 8. 数据设计结论

- 本次改造的数据重点不仅是实体映射和双语文案，还包括中文 Demo 的 IoT 依赖清理、英文 Demo 配置样例、mock 数据集和 MCP 双语样例。
- 会议纪要和参考图对应的数据输入只需保留必要摘要，用于解释需求来源和范围边界，不扩展为新的大规模数据建设任务。
- 用于识别当前问题的数据必须真实梳理，用于最终演示运行的数据应优先 mock 化。
- 英文能力和演示稳定性最终取决于别名映射、指令映射、mock 数据和双语样例是否完整。
- 阶段二开发前，至少应先完成 IoT 依赖清单、英文 Demo 配置样例、mock 数据样例、可选字段缺省样例和 MCP 双语样例。
