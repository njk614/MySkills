## 1. 文档目的

本文档用于根据功能需求设计、技术需求设计和数据需求设计，建立可执行、可复现、可量化判定的验收测试用例集，作为阶段二开发完成后的统一验收依据。

## 3. 验收前置条件

### 3.1 环境前置条件

- 存在一个可用的测试 token，且对应场景数据可正常查询。
- 中文 Demo 已完成孪生体梳理和去真实 IoT 化改造。
- 英文 Demo 已完成基础配置。
- 模拟数据生成器已可提供传感器数据和操作接口回执。
- command、dataquery、opeationrule、spacecount 四类 skill 对应脚本可正常调用。
- 孪易产品前后端已接入英文指令处理链路。
- MCP 接口已提供中英文调用入口和双语文档样例。
- 演示环境默认已切换到 mock 数据模式，除专项验证外不以 live 数据作为前置条件。

### 3.2 数据前置条件

- 至少有 1 组可用温度设备数据，可来自真实或 mock。
- 至少有 2 个可查询的摄像头实例名称。

- 至少有 1 条英文告警规则样例。
- 至少有 1 组可查询的空间区域结果。
- 至少有 1 组 pending 待确认样例。

- 至少有 1 组 MCP 中英文请求与返回样例。

- 至少有 1 组未显式传 data_source_mode 的 Demo 请求样例。

## 6. 通过标准

### 6.1 单用例通过标准

- 前置条件可复现。
- 输入步骤清晰且可执行。
- 返回结果与预期结果一致。
- 成功或失败标准可量化，不使用主观描述。

### 6.2 整体验收通过标准

- 所有 P0 用例全部通过。
- 中文回归用例必须全部通过。
- 中文 Demo 去真实 IoT 依赖验证必须通过。
- 英文 Demo 配置验证必须通过。
- 孪易英文指令一致性和 MCP 双语接口兼容用例必须通过。
- locale 缺省兼容和 data_source_mode 缺省 mock 用例必须通过。
- 若页面展示未完全复刻参考图，但现有 Demo 上的双语能力、mock 化和演示闭环已达成，不应判定为不通过。

### 6.3 不通过判定规则

- 英文请求触发到错误 skill 或无法触发 skill。
- 英文返回仍为主要中文文案，且非后端真实字段回传。
- Demo 运行仍依赖真实 IoT。
- 英文 Demo 只有文案变化，无法完成真实操作演示。
- 英文指令与中文效果不一致。
- MCP 中英文调用或返回任一链路不兼容。
- 缺少 locale 或 data_source_mode 可选字段时主链路直接失败。

## 7. 验收建议执行方式

### 7.1 执行顺序

1. 先执行中文 Demo 去 IoT 化与 mock 数据接管验证。
2. 再执行英文查询与英文执行主链路。
3. 再执行英文确认和取消与规则联动。
4. 再执行英文 Demo 配置、孪易英文指令一致性和 MCP 双语接口验证。
5. 再执行 locale 和 data_source_mode 缺省兼容测试。
6. 最后执行中文回归和中英文共存测试。

### 7.2 建议记录项

- 测试时间
- 测试环境
- 数据源模式（mock 或 live）
- 输入文本或操作步骤
- 返回结果或截图
- 是否通过
- 失败原因与归属模块

## 9. 用例设计结论

- 本次验收测试用例已覆盖阶段一确认的全部 P0 路径。
- 验收重点不再只有 Skill 双语能力，还覆盖 Demo 去 IoT 化、英文 Demo 配置、mock 数据接管、英文指令一致性和 MCP 双语接口兼容。
- 验收还应覆盖 locale、data_source_mode 等可选字段在缺省情况下的兼容行为，避免接口改造后出现隐藏回归。
- 华为会议纪要与参考图在验收层面仅作为对标背景，不作为强制重做式外观验收基线。
- 因此，本文档可直接作为完整 Demo 双语改造项目的阶段二验收依据。
