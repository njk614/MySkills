# 指令规则

## 目标

将用户自然语言转换为两份结果：

1. 执行用指令串：保留 `A/B/C/D` 编码，用于发送执行 HTTP 请求。
2. 展示用中文文本：不保留编码，只保留用户可理解的动作说明。

## 查询类指令

- `D01` 用于查询类结果。
- 展示时不再保留 `D01` 编码。
- 直接输出“为您查找到相关内容如下：...”的自然语言文本。

## 展示文本规则

### 原则

- 用户看到的结果不带编码。
- HTTP 执行的 `instruction_order` 保留编码。
- 不要把执行编码直接展示给用户。

### 示例

- `A09` → 展示 `场景复位`
- `A08：场景旋转：开始` → 展示 `场景旋转：开始`
- `A01：功能切换：告警` → 展示 `功能切换：告警`
- `B07：打开灯：1F走廊灯` → 展示 `打开灯：1F走廊灯`
- `C02：主题生成：园区能耗分析` → 展示 `主题生成：园区能耗分析`

## 多指令规则

- 多条执行指令使用 `&` 连接。
- 展示时按序号拼接：
  - `1、...`
  - `2、...`
- 非查询类统一前缀：
  - `根据最优策略，已经为您规划如下执行计划：`

## 未匹配规则

遇到以下情况时，不要伪造结果：

- 名称类参数未匹配到。
- 状态类参数未判断出。
- 只返回裸编码但该指令必须带参数。

此时应明确返回失败原因或候选信息。
